## Задача
Instance segmentation процесса индукционной пайки волноводов с помощью **YOLO11-seg**.
- **3 класса сегментации**: waveguide (0), flux (1), solder (2)
- **4 этапа техпроцесса** определяются по динамике масок:
  - 0: Предварительный нагрев
  - 1: Плавление флюса (~300°C)
  - 2: Плавление припоя (~580-600°C)
  - 3: Стабилизация

## КРИТИЧЕСКАЯ ПРОБЛЕМА: Аннотации Roboflow неправильные

Оригинальные аннотации из Roboflow **содержат неточности в границах объектов**. Пользователь (Danny) предоставил 3 эталонных ручных разметки — файлы в `results/`:

### Что есть что на кадре

На каждом кадре видна **установка для индукционной пайки** — большой металлический корпус (индуктор) с углублением. В центре углубления находятся объекты пайки:

#### Waveguide (класс 0)
- **МАЛЕНЬКИЙ прямоугольный металлический компонент** в центре кадра
- Состоит из: **тёмная прямоугольная полость** (чёрное отверстие сверху) + **тонкие металлические стенки** вокруг неё (светлые/серебристые края)
- Это сам волновод — металлическая трубка прямоугольного сечения
- Размер примерно 80-120px в кадре 640x640
- Эталон: `results/user_annotation_waveguide.jpg` — жёлтый контур = стенки, фиолетовый = полость
- **ВАЖНО**: waveguide — это ТОЛЬКО маленький прямоугольник, НЕ весь корпус индуктора!

#### Flux (класс 1)
- **Маленькие яркие белые/голубоватые пятна** НЕПОСРЕДСТВЕННО примыкающие к ЛЕВОЙ и ПРАВОЙ сторонам waveguide
- Это флюс, нанесённый на боковые поверхности
- Эталон: `results/user_annotation_flux.jpg` — бирюзовые пятна по бокам

#### Solder (класс 2)
- **ТОНКАЯ U-образная металлическая полоска** обнимающая НИЖНИЙ край и нижние стороны waveguide
- Это припой, лежащий у основания волновода
- Эталон: `results/user_annotation_solder.jpg` — зелёная U-форма снизу

### Типичные ошибки предыдущих попыток
1. Waveguide размечен слишком большим — захватывает фон, корпус индуктора
2. Flux перекрывается с waveguide
3. Solder слишком толстый или не в том месте
4. Контуры грубые (прямоугольники вместо точных границ)

## Что было сделано (история Cowork-сессии)

### Попытки автоматической переразметки (ВСЕ НЕУДАЧНЫЕ, файлы удалены):
- **v2**: Простой threshold → слишком грубо
- **v3**: Edge detection + morphology → waveguide слишком большой ("ты сильно вылазишь за контуры")
- **v4**: Tighter expansion + edge refinement → всё ещё не так
- **my_vision.ipynb**: Ручная попытка разметить координаты → "ты косишь жёстко" (полностью мимо)
- Все скрипты reannotate*.py, auto_annotate_waveguide.py, и датасеты dataset_v2/v3/v4 **удалены** при чистке проекта

### Тестирование 7 методов сегментации (текущее состояние):
Создан `annotation_methods.ipynb` — сравнение 7 подходов на 4 тестовых кадрах:
1. **Dark Cavity** — threshold на тёмную полость → expand на стенки → слишком мало, только полость
2. **GrabCut** — ловит стенки + полость, но захватывает лишний фон
3. **K-means** — цветовая кластеризация → слишком большая область
4. **Canny + Rectangle** — edge detection → неправильные контуры
5. **Watershed** — нестабильный, часто на фоне
6. **CLAHE + Otsu** — рассыпается
7. **Bright + Dark Hull** — слишком большой convex hull

Результаты визуализации: `results/MVI_6265_methods.jpg`, `results/MVI_6265_overlay.jpg` и т.д.

**Ни один метод не дал идеального результата.** Пользователь ещё не выбрал лучший.
